## Contribution Philosophy

This repository is designed for clarity, reproducibility, and long-term maintainability. Contributions should:

- Preserve modular structure (`src/`, `configs/`, `docs/`)
- Include meaningful commit messages
- Be accompanied by documentation updates when relevant
- Respect type safety and fallback logic conventions

---

## How to Contribute

If you'd like to contribute:

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-enhancement`)
3. Make changes with clear commit messages
4. Submit a pull request with a summary of changes
